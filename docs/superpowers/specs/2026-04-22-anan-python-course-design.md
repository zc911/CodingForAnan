# 设计文档：Anan 的 Python 编程课程网站

**日期：** 2026-04-22  
**目标用户：** 9岁（小学三年级）女生，有 Scratch 基础  
**课程规模：** 14节课 + 3个阶段项目，每节约1小时

---

## 一、项目目标

为女儿 Anan 定制一个 Python 编程学习网站，支持自学（家长在旁辅助）。学完后她能：
1. 独立编写 50-100 行的 Python 小程序
2. 建立系统的计算机科学基础认知（变量、控制流、数据结构、算法思维）

---

## 二、技术架构

### 技术栈

| 层次 | 技术 | 用途 |
|------|------|------|
| 内容 | Markdown | 每节课的文字内容 |
| 构建 | Python 脚本 + Jinja2 | 将 Markdown 编译为静态 HTML |
| 交互代码 | Pyodide + CodeMirror | 浏览器内运行 Python，语法高亮编辑 |
| 样式 | 原生 CSS（自定义） | 儿童友好视觉风格 |
| 部署 | Cloudflare Pages | 推送 Git 自动部署 |

### 目录结构

```
CodingForAnan/
├── content/              # 每节课的 Markdown 源文件
│   ├── 01-hello/
│   │   ├── lesson.md     # 课程内容（含代码块元数据）
│   │   └── exercises/    # 本地练习 .py 文件
│   ├── 02-variables/
│   └── ...
├── templates/            # Jinja2 HTML 模板
│   ├── base.html         # 通用布局（导航、Pyodide 加载）
│   ├── lesson.html       # 单节课页面
│   └── index.html        # 主页课程目录
├── static/               # 静态资源
│   ├── style.css
│   ├── editor.js         # Pyodide + CodeMirror 封装
│   └── images/
├── build.py              # 构建脚本
└── dist/                 # 生成的静态文件（Cloudflare Pages 部署目录）
```

### 构建流程

```
content/*.md  →  build.py (Jinja2)  →  dist/*.html  →  Cloudflare Pages
```

本地预览：`python build.py`，在浏览器打开 `dist/index.html`。

## 三、课程大纲（14节 + 3个项目）

### 阶段一：认识 Python（第1-3节）

| 节次 | 主题 | 实践内容 |
|------|------|----------|
| 01 | 计算机是怎么工作的？认识 Python | 运行第一行 `print("Hello, Anan!")` |
| 02 | 变量和数据类型（`int`, `str`, `float`, `bool`） | 用变量存自己的名字、年龄、身高 |
| 03 | 输入与输出（`input()` / `print()`） | 写一个"自我介绍小程序" |

### 阶段二：控制流程（第4-6节）+ 项目一

| 节次 | 主题 | 实践内容 |
|------|------|----------|
| 04 | 条件判断（`if` / `elif` / `else`） | 判断成绩等级、判断奇偶数 |
| 05 | 循环（`while`） | 倒计时程序、累加求和 |
| 06 | 循环（`for` + `range`） | 打印九九乘法表 |
| **P1** | **🎮 项目：猜数字游戏** | 电脑随机出数，玩家猜，有"大了/小了"提示 |

### 阶段三：组织数据与代码（第7-10节）+ 项目二

| 节次 | 主题 | 实践内容 |
|------|------|----------|
| 07 | 列表（`list`） | 购物清单、成绩排名 |
| 08 | 函数（`def`） | 把猜数字游戏重构为函数 |
| 09 | 字符串操作 | 倒着念名字、统计字母数 |
| 10 | 模块与 `import`（`random`, `math`） | 用 `random` 做随机出题程序 |
| **P2** | **🎨 项目：turtle 绘图** | 用 `turtle` 画一幅画或写自己名字 |

### 阶段四：进阶与综合（第11-14节）+ 项目三

| 节次 | 主题 | 实践内容 |
|------|------|----------|
| 11 | 字典（`dict`） | 单词本、通讯录 |
| 12 | 文件读写（`open`） | 把笔记存到文件里 |
| 13 | 错误处理（`try` / `except`） | 让程序不容易崩溃 |
| 14 | CS 基础：算法思维（排序/查找） | 可视化演示冒泡排序 |
| **P3** | **🛠️ 项目：自选小程序** | 从3个选题里选一个完成 |

**每节课结构：** 理论讲解（~15分钟）→ 浏览器演示代码（~20分钟）→ 本地练习题（~25分钟）

## 四、交互式代码编辑器

### 两种代码块

**演示代码块（Demo Block）：**
- 基于 CodeMirror，有语法高亮，代码可编辑
- 点击"▶ 运行"通过 Pyodide 在浏览器执行
- 输出区显示 `print` 结果；报错时显示友好中文提示

**练习题块（Exercise Block）：**
- 提供填空式起始代码（降低空白页焦虑）
- "💾 下载到 Thonny"按钮将当前代码下载为 `.py` 文件
- 较复杂的练习只提供下载，不在浏览器中完成

### Pyodide 加载策略

- 进入课程页面时**后台预加载** Pyodide（~10MB）
- 加载中显示进度条，"运行"按钮置灰直到加载完成
- 同一会话内所有代码块共享一个 Pyodide 实例，不重复加载

### input() 的处理

Pyodide 不支持原生阻塞式 `input()`。涉及 `input()` 的浏览器演示块采用以下方式：在代码运行前，页面上方出现一个输入框区域，让用户预先填写 `input()` 的返回值（类似"模拟输入"）。演示后附注说明：正式练习在 Thonny 里运行，才能体验真正的交互输入。

## 五、页面设计

### 主页
- 展示4个阶段、14节课 + 3个项目
- 每节课卡片：编号、标题、简短描述、完成状态
- 完成状态用 `localStorage` 本地保存，无需登录

### 每节课页面布局

```
顶部导航栏
  ← 上一课         第N课：标题         下一课 →

正文区（宽屏双栏）
  左栏（60%）：课程内容 + 代码块
  右栏（40%）：本节知识点速查卡（sticky 固定）

底部
  ✅ 标记本课完成        下载本课所有练习
```

### 视觉风格
- **字体：** 正文 18-20px，代码区等宽字体
- **配色：** 主色蓝/绿，代码区深色背景
- **语法高亮：** 关键字橙色、函数名蓝色、字符串绿色
- **图示：** SVG 概念图（如变量 = 贴标签的盒子）
- **正向反馈：** 代码运行成功时偶尔出现小庆祝动画
- **语言：** 中文讲解，代码和专业术语使用英文

### 响应式
- 主要面向 iPad / 桌面使用
- 移动端做单栏基本适配，代码编辑体验以桌面为主

## 六、内容格式约定

课程 Markdown 文件通过特定代码块标记区分演示和练习：

````markdown
```python demo title="打印你的名字"
name = "Anan"
print("你好，" + name + "！")
```

```python exercise title="改成打印你自己的名字"
# 把下面的名字改成你自己的名字
name = "___"
print("你好，" + name + "！")
```
````

构建脚本解析这些标记，生成对应的交互式 HTML 组件。
